# Project Rules

## PR 创建方式
- 创建 PR 时，始终使用浏览器方式（打开 GitHub PR 创建页面链接），不使用 `gh` CLI
- PR 创建链接格式：`https://github.com/Miles128/SellWise/compare/{base}...{head}`
- 默认 PR 方向：从功能分支合并到 main 分支

## 后端开发环境
- 后端使用 `uv` 作为 Python 包管理器和虚拟环境
- 运行后端命令前，必须进入 `backend` 目录并使用 `uv run` 前缀
- 正确的运行方式：
  ```bash
  cd backend
  uv run python run.py
  uv run python <script.py>
  uv sync  # 同步依赖
  ```
- 不要直接使用系统 Python (`python3`) 或 `pip` 安装依赖
- 依赖配置文件：`backend/pyproject.toml`

## 前端开发环境
- 前端使用 `npm` 作为包管理器
- 运行命令：`npm run dev`（开发模式）、`npm run build`（构建）

## 项目功能规划

### 1. 选品环节
- **核心流程**：用户输入大品类关键词 → 自动化搜索工具获取数据 → AI 大模型分析 → 返回热点品类推荐
- **技术需求**：
  - 自动化搜索工具（爬虫/Selenium/API 集成）
  - AI 大模型集成（热点判断、趋势分析）
  - 热点指标定义（搜索量、趋势斜率、竞争度等）

### 2. 商品运营数据分析
- **说明**：合并工作台概览和运营数据分析功能，提供一站式商品运营数据查看
- **数据导入方式**：CSV/Excel 文件上传
- **核心功能模块**：
  - 总概况图（销售趋势、品类分布、热销产品排行等）
  - 商品细节趋势图（单个商品的销量、库存、价格趋势）
  - 款型对比分析（多维度对比）
  - **规划功能**：品类相关性计算（统计分析、关联规则挖掘）
- **技术需求**：
  - CSV/Excel 文件解析（支持多格式、多表）
  - 数据可视化库（图表生成）
  - 统计学算法库（相关性计算）

### 3. 小红书达人运营板块
- **数据导入方式**：CSV/Excel 文件上传
- **数据关系模型**：达人 (1:N) 帖子 (1:1) 商品
- **核心功能模块**：
  - 达人运营数据计算（粉丝数、互动率、转化率等）
  - 商品运营数据计算（销量、曝光、转化等）
  - 帖子绩效打分（基于多维度指标）
  - 达人整体打分（综合所有帖子绩效）
- **技术需求**：
  - 多表关联数据模型设计
  - 打分算法设计（权重分配、指标归一化）
  - 数据更新时的分数重算机制
